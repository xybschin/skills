# Skills

Skills organized by workflow stage.

## Categories

| Category | Purpose | When to Use |
|----------|---------|------------|
| **planning** | Convert ideas into issues and PRDs | After design is solid, ready to plan implementation |
| **implementation** | Write and improve code and skills | During development, refactoring, or creating new skills |
| **process** | Manage issues and commits | Managing backlog and committing changes |
| **thinking** | Stress-test designs with rigorous questioning | Before committing to a plan or major decision |

## Getting Started

Start with the category that matches your current workflow stage, then explore individual skill READMEs within each directory.

**Note**: Several skills require `setup-skills` to be run first in your repository. See the `planning/` category README for details.
